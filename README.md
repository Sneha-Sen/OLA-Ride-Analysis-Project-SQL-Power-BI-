# OLA-Ride-Analysis-Project-SQL-Power-BI-
This project analyzes OLA ride booking data using MySQL for data exploration and Power BI for dashboard visualization.
It provides insights into booking patterns, cancellations, revenue distribution, and customer behavior.

# Objective

To perform data-driven analysis of OLA’s operational data and create an interactive Power BI dashboard that helps understand:
- How customers book rides
- Which payment methods generate more revenue
- What factors lead to cancellations
- Trends in ride distances and booking volumes

# Tools & Technologies

- MySQL for Data extraction, cleaning, and analysis through SQL queries
- Power BI for Visualization and dashboard creation
- Excel/CSV	- Used as the data source after SQL processing

# Skills Demonstrated

- SQL
- Data Cleaning and Transformation in Power BI
- DAX Measures and Calculated Columns
- Dashboard Design & Data Storytelling
- Business Analysis & Interpretation of KPIs

# Power BI Dashboard Highlights

-- KPIs:

Total Bookings – 20,407

Total Booking Value – ₹7M

Successful Bookings – 12,652

Cancelled Bookings – 5,735

Cancellation Rate – 28.10%

-- Visuals Included:

- Top 5 Vehicle Types by Ride Distance
- Revenue by Payment Method
- Booking Status Breakdown (Success vs Cancellations by Reason)
- Ride Distance Distribution per Day
- Ride Volume Over Time
- Top Customers by Booking Value

# Insights Gained

- Cash is the most preferred payment method (₹3.8M revenue).
- Bike and eBike rides dominate by distance covered.
- Cancellation rate (28%) is high, mainly due to driver-related issues and plan changes.
- Ride volume fluctuates around weekends, peaking around mid and end of the month.
